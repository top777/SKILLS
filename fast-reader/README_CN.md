
# Fast Reader: 智能文档高效解析引擎

##  简介
**Fast Reader** 是一款专为 Claude Code 打造的阅读辅助技能（Skill）。它将人类的“结构化快速阅读技巧”转化为 AI 的底层执行逻辑，专为处理代码库、系统日志、产品说明书、项目计划书及测试报告等大型文档而设计。

通过引入**渐进式披露（Progressive Disclosure）**与**按需检索**机制，Fast Reader 能够精准定位高价值信息，在确保不丢失核心重点的前提下，最大程度节省上下文（Context Window）的 Token 消耗。

##  核心特性
- ** 结构化侦察**：拒绝从头到尾线性阅读。优先提取文档骨架（首尾、目录、标题），快速建立全局认知。
- ** 场景化扫描**：
  - **代码文件**：自动过滤样板代码与依赖导入，直击核心业务逻辑与导出接口。
  - **系统日志**：利用 `Grep` 精准拦截 `ERROR`、`Exception` 等关键异常，自动忽略常规 `INFO` 噪音。
  - **长文档/报告**：遵循 80/20 法则，聚焦摘要、结论、风险评估与行动项，跳过冗长背景铺垫。
- ** 极致省流**：通过工具调用限制与结构化输出约束，大幅降低读取与生成的 Token 开销。
- ** 结构化输出**：强制采用“核心摘要 ️ 关键发现 ️ 风险/行动项”的标准模板，杜绝 AI 废话与原文摘抄。

##  快速开始
1. 将技能文件保存至你的项目目录：`.claude/skills/fast-reader/SKILL.md`
2. 在 Claude Code 中直接使用以下指令触发：
   - `/fast-reader 分析 src/core/engine.ts 的核心逻辑`
   - `/fast-reader 总结昨天的部署日志，找出所有报错`
   - `帮我快速阅读这份产品需求说明书，提炼核心功能点`

## ️ 适用场景
- 接手遗留代码，需要快速摸清项目架构
- 排查生产事故，需要在海量日志中定位根因
- 审阅几十页的技术方案或测试报告，提取核心结论
